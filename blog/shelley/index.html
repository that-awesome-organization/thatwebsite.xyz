    Shelley

    Shelley is a mobile-friendly, web-based, multi-conversation, multi-modal, multi-model, single-user coding agent built for but not exclusive to exe.dev.

    This post is more about my thoughts on Shelley a coding agent, how I have set it up for my workflows.

    The “Doer” Attitude

    Shelley is more than an LLM interface; it’s a persistent development environment. When I ask it to “Add a new feature to the theme,” it doesn’t just give me the code. It:

    1. Navigates the filesystem to find the relevant files.
    2. Analyzes the existing code to match the code patterns.
    3. Applies the change using patch.
    4. Starts the development server and checks if the site is up and works after build or any test cases if created.
    5. Commits the final work to Git with a meaningful message.

    This “Observe -> Think -> Act” loop turns a static AI into an active participant in the codebase.

    Navigation and Analysis

    Unlike a standard chatbot that only generates code blocks, Shelley is designed for execution, like most of the other Coding Agents

    Edits with Diffs

    It uses tools like patch for precise file modifications instead of full-file rewrites.

    Also each of the change the agent does in the conversation, it is been committed, so even a smallest change can be tracked, and a very intutive way to comment and ask for a reason for change or ask for any other approach.

    Integrated Terminal: Full bash access to run hugo server, install dependencies, or execute complex shell scripts directly.
Git Native: Automatically making focused commits with descriptive messages, turning the development process into a clean version-controlled history.
